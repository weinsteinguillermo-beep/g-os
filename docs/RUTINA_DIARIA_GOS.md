# Rutina Diaria G-OS

## Proposito

Esta guia explica como usar G-OS en la primera prueba real desde el movil.

El objetivo no es cargar todo. El objetivo es que Guillermo pueda abrir G-OS, decidir foco, capturar ideas y cerrar el dia con continuidad.

## Manana

Tiempo objetivo: menos de 2 minutos.

1. Abrir `app/index.html` desde el movil.
2. Usar `Modo manana`.
3. Leer la recomendacion del dia.
4. Revisar solo:
   - briefing;
   - decisiones;
   - proyectos clave;
   - nueva idea si aparece algo;
   - mision Codex si hay tarea para preparar.
5. Aprobar, seguir analizando o archivar decisiones.

Regla: si una decision no cambia el foco del dia, no debe consumir tiempo.

## Durante El Dia

Usar G-OS solo para:

- capturar una idea;
- ajustar una decision;
- revisar un proyecto clave;
- generar una mision para Codex;
- exportar datos si se quiere respaldo.

No usar G-OS como lista de tareas general.

## Cierre Del Dia

Tiempo objetivo: menos de 3 minutos.

1. Usar `Modo cierre`.
2. Revisar:
   - decisiones tomadas;
   - ideas capturadas;
   - pendientes para manana;
   - que debe preparar Codex.
3. Escribir el aprendizaje del dia.
4. Guardar aprendizaje.
5. Tocar `Copiar cierre del dia`.
6. Pegar el cierre en ChatGPT o WhatsApp si se quiere continuar desde otro lugar.

## Que No Cargar Todavia

No cargar:

- todos los clientes;
- todos los productos;
- todas las tareas;
- historiales largos;
- documentos completos;
- datos sensibles;
- automatizaciones;
- integraciones;
- informacion que no ayude a decidir.

## Criterio De Exito De La Prueba

La prueba funciona si Guillermo abre G-OS manana y puede responder:

- que decision importa hoy;
- que proyecto requiere atencion;
- que idea conviene guardar;
- que debe preparar Codex;
- que aprendizaje queda para manana.

## Regla Final

G-OS se prueba como rutina, no como sistema completo.
