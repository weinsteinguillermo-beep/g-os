# User Flow

## Flujo Principal

Guillermo abre G-OS desde el iPhone.

1. Ve el Daily Briefing.
2. Identifica decisiones, riesgos y oportunidades.
3. Aprueba, posterga o pide mas analisis.
4. Captura ideas si aparecen.
5. Revisa proyectos solo si necesita contexto.
6. Mira Codex solo para saber que esta trabajando.

## Flujo Daily Briefing

Entrada:

- abrir app.

Salida:

- una recomendacion del dia;
- decisiones visibles;
- claridad sobre foco.

## Flujo Nueva Idea

Entrada:

- tocar Nueva Idea;
- escribir idea;
- guardar.

Salida:

- idea capturada;
- sin clasificacion;
- sin compromiso de ejecucion.

## Flujo Decisiones

Entrada:

- revisar decision.

Acciones:

- Aprobar;
- Seguir analizando.

Salida:

- estado local actualizado;
- proximo paso claro.

## Flujo Codex

Entrada:

- revisar trabajos;
- tocar Nueva mision.

Salida:

- preparacion simulada de una nueva mision;
- futuro enlace con template de prompt.

## Regla De Experiencia

Cada flujo debe terminar con claridad: aprobado, capturado, pendiente o listo para preparar.
