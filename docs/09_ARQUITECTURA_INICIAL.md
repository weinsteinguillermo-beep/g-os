# Arquitectura Inicial De G-OS

## Enfoque

La arquitectura inicial de G-OS es documental. Primero se ordena el conocimiento. Luego se definen bases, automatizaciones, dashboards e integraciones.

## Componentes Iniciales

### 1. Repositorio Documental

Lugar donde viven principios, modelos, agentes, proyectos, plantillas y decisiones arquitectonicas.

### 2. Centro De Proyectos

Fichas por empresa, proyecto o linea de trabajo. Cada ficha debe registrar contexto, estado, prioridad, proximas acciones y decisiones pendientes.

### 3. Registro De Ideas

Sistema simple para capturar, clasificar y evaluar ideas.

### 4. Registro De Decisiones

Modelo estandar para registrar decisiones importantes y aprender de ellas.

### 5. Agentes IA

Roles especializados para preparar trabajo, analizar contexto y proponer acciones.

### 6. Integraciones Futuras

Herramientas previstas:

- ChatGPT;
- Codex;
- Gmail;
- Google Drive;
- Airtable;
- n8n;
- WhatsApp;
- Power BI;
- Excel;
- OneDrive.

## Principio Arquitectonico

No automatizar un proceso que todavia no se entiende.

Primero se documenta el flujo real. Despues se estructura. Luego se automatiza.

## Identificadores Futuros

Cuando el sistema pase a bases de datos, cada entidad debera tener identificador unico:

- `IDEA-0001`;
- `PROY-0001`;
- `DEC-0001`;
- `CLI-0001`;
- `AGT-0001`;
- `ALR-0001`.

## Proxima Evolucion

La siguiente etapa debe definir donde vivira la fuente de verdad operativa: documentos, Airtable, Drive, OneDrive o una combinacion controlada.
