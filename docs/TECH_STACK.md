# Tech Stack

## Stack Elegido Para El MVP

- HTML.
- CSS.
- JavaScript.
- LocalStorage para simulacion local.

## Motivo

El MVP debe validar experiencia, no demostrar tecnologia.

HTML, CSS y JavaScript permiten:

- carga muy rapida;
- uso sin servidor;
- compatibilidad movil;
- facil mantenimiento;
- cero dependencias;
- bajo riesgo.

## Que No Se Usa Todavia

- React.
- Next.js.
- Tailwind.
- Backend.
- Base de datos.
- Autenticacion.
- API externa.

## Camino Futuro

Si el MVP funciona, la arquitectura puede evolucionar a:

- React o Next.js para componentes escalables;
- API liviana para persistencia;
- Airtable como base operativa inicial;
- n8n para automatizaciones;
- integraciones con Gmail, Drive y Codex;
- Power BI para tablero ejecutivo.

## Regla

La tecnologia debe seguir a la experiencia, no al reves.
