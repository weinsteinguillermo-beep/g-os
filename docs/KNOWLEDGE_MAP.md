# Knowledge Map De G-OS

## Proposito

Este documento define como se relaciona el conocimiento dentro de G-OS. Su objetivo es que futuros agentes encuentren contexto automaticamente y eviten trabajar con informacion aislada.

## Entidades Principales

### Empresas

Representan unidades de negocio, sociedades, marcas o lineas empresariales.

Se relacionan con:

- clientes;
- proyectos;
- productos;
- proveedores;
- documentos;
- correos;
- reuniones;
- decisiones;
- indicadores;
- resultados.

### Clientes

Representan personas, empresas u organizaciones con relacion comercial actual o potencial.

Se relacionan con:

- empresas que los atienden;
- oportunidades;
- productos de interes;
- correos;
- reuniones;
- propuestas;
- decisiones comerciales;
- resultados de venta;
- aprendizajes de relacion.

### Proyectos

Representan iniciativas con objetivo, alcance, estado y proximas acciones.

Se relacionan con:

- empresa responsable;
- ideas que los originaron;
- documentos;
- agentes IA;
- decisiones;
- reuniones;
- clientes;
- proveedores;
- productos;
- indicadores;
- resultados;
- aprendizajes.

### Documentos

Se relacionan con:

- proyectos;
- empresas;
- decisiones;
- reuniones;
- aprendizajes;
- agentes que los crearon o actualizaron.

- reunion;
- proxima accion;

## Relaciones Clave

## Idea A Proyecto

Una idea puede originar un proyecto si supera evaluacion de oportunidad, prioridad y capacidad de ejecucion.

## Proyecto A Decision

Todo proyecto activo deberia tener decisiones registradas cuando existan cambios de alcance, inversion, prioridad, cliente, proveedor o estrategia.

## Decision A Resultado

Toda decision importante debe tener un resultado esperado y, luego, un resultado real.

## Resultado A Aprendizaje

Todo resultado relevante debe generar aprendizaje para no repetir errores ni perder patrones valiosos.

## Cliente A Oportunidad

Cada cliente puede estar vinculado a oportunidades actuales, historicas o futuras.

## Documento A Contexto

Cada documento debe indicar a que empresa, proyecto, decision o aprendizaje pertenece.

## Agente A Trazabilidad

Cada agente debe dejar claro que analizo, que supuso, que recomendo y que queda pendiente.
