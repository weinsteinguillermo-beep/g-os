# Learning System De G-OS

## Proposito

El Learning System es el sistema de memoria empresarial de G-OS. Su funcion es transformar experiencia en conocimiento reutilizable.

## Principio

G-OS aprende cuando conecta lo que ocurrio con lo que se decidio, lo que resulto y lo que conviene hacer diferente la proxima vez.

## Que Registrar

## Que Ocurrio

Descripcion breve del hecho, evento, decision, venta, reunion, problema u oportunidad.

Debe incluir:

- fecha;
- empresa;
- proyecto;
- personas involucradas;
- fuente;
- contexto.

## Que Funciono

Registrar acciones, argumentos, procesos, productos, personas o decisiones que dieron resultado.

Preguntas utiles:

- Que genero avance?
- Que facilito la decision?
- Que aumento margen, velocidad o claridad?
- Que deberia repetirse?

## Que No Funciono

Registrar errores, fricciones, demoras, falsas suposiciones o acciones sin impacto.

Preguntas utiles:

- Que bloqueo avance?
- Que informacion falto?
- Que se asumio mal?
- Que costo tiempo o dinero?
- Que no deberia repetirse?

## Que Aprendimos

Convertir experiencia en regla practica.

Un buen aprendizaje debe poder escribirse asi:

Si ocurre X, conviene hacer Y, porque aprendimos Z.

## Que Patrones Aparecen

Detectar repeticiones entre proyectos, clientes, proveedores, productos o decisiones.

Tipos de patrones:

- comerciales;
- operativos;
- financieros;
- logisticos;
- humanos;
- tecnologicos;
- de mercado.

## Como Reutilizar El Aprendizaje

Cada aprendizaje debe poder convertirse en:

- recomendacion futura;
- alerta;
- checklist;
- regla de decision;
- mejora de proceso;
- pregunta para negociacion;
- criterio de prioridad.

## Ciclo De Aprendizaje

1. Registrar hecho.
2. Vincularlo a decision, proyecto o cliente.
3. Comparar resultado esperado y real.
4. Extraer aprendizaje.
5. Detectar patron.
6. Actualizar recomendacion futura.
7. Recuperar aprendizaje antes de decisiones similares.

## Formato Minimo De Aprendizaje

- Titulo:
- Fecha:
- Empresa:
- Proyecto:
- Que ocurrio:
- Que funciono:
- Que no funciono:
- Aprendizaje:
- Patron detectado:
- Como reutilizarlo:
- Documento o decision relacionada:

## Regla

Un aprendizaje que no puede reutilizarse todavia es una observacion. Debe guardarse, pero no debe tratarse como regla definitiva.
