# Components

## Principio Visual

G-OS debe sentirse como Notion mezclado con Apple:

- mucho espacio;
- pocos colores;
- textos claros;
- jerarquia simple;
- sin efectos innecesarios;
- orientado a decision.

## Componentes Del MVP

## App Shell

Contenedor principal de la experiencia movil.

Debe incluir:

- encabezado;
- fecha;
- estado del dia;
- navegacion por modulos;
- contenido principal.

## Daily Briefing Panel

Panel inicial y principal.

Incluye:

- saludo;
- decisiones;
- riesgos;
- oportunidades;
- agenda;
- Codex trabajando;
- recomendacion del dia.

## Quick Idea Capture

Modulo para capturar ideas rapidamente.

Incluye:

- boton grande;
- campo de texto;
- accion Guardar idea;
- lista corta de ideas capturadas.

## Project List

Lista ejecutiva de proyectos.

Incluye:

- nombre;
- estado;
- prioridad;
- ultima actividad.

## Decision Card

Tarjeta simple para decisiones.

Incluye:

- contexto;
- recomendacion;
- acciones: Aprobar, Seguir analizando.

## Codex Work Panel

Modulo de trabajos de Codex.

Incluye:

- trabajos en curso;
- ultima ejecucion;
- boton Nueva mision.

## Empty State

Mensaje breve cuando no hay datos.

Debe orientar a accion, no explicar el sistema.
