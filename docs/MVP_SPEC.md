# MVP Spec De G-OS

## Objetivo

Construir el primer producto real de G-OS: una aplicacion web extremadamente simple, rapida y usable desde iPhone.

El objetivo principal es que Guillermo quiera abrir G-OS todas las mananas.

## Alcance Del MVP

El MVP valida experiencia, no infraestructura.

Incluye cinco modulos:

1. Daily Briefing.
2. Nueva Idea.
3. Proyectos.
4. Decisiones.
5. Codex.

## Fuera De Alcance

- APIs.
- Base de datos.
- Login.
- Integraciones con Gmail, Drive, Airtable, n8n o Power BI.
- Automatizaciones.
- Panel administrativo.
- Sistema completo de agentes.

## Criterio De Exito

Guillermo debe poder abrir la app desde el movil y entender en menos de dos minutos:

- que decisiones requieren atencion;
- que riesgos importan;
- que oportunidades hay;
- que proyectos se movieron;
- que esta haciendo Codex;
- cual es la recomendacion del dia.

## Modulo 1: Daily Briefing

Debe mostrar:

- Buenos dias Guillermo.
- Decisiones.
- Riesgos.
- Oportunidades.
- Agenda.
- Codex trabajando.
- Recomendacion del dia.

Debe sentirse como un briefing ejecutivo, no como un reporte.

## Modulo 2: Nueva Idea

Debe incluir un boton enorme y un campo simple para escribir una idea.

No clasifica todavia. Solo captura.

## Modulo 3: Proyectos

Debe mostrar:

- nombre;
- estado;
- prioridad;
- ultima actividad.

## Modulo 4: Decisiones

Cada decision muestra:

- contexto;
- recomendacion;
- boton Aprobar;
- boton Seguir analizando.

En el MVP, los botones solo simulan estado local.

## Modulo 5: Codex

Debe mostrar:

- trabajos en curso;
- ultima ejecucion;
- boton Nueva mision.

## Datos

Todos los datos son simulados.

La captura de ideas puede guardarse localmente en el navegador para validar sensacion de uso, sin considerarse fuente de verdad.
