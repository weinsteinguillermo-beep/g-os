# Modelo Operativo De G-OS

## Objetivo

Definir como G-OS organiza informacion, contexto, decisiones, proyectos y agentes para convertir actividad dispersa en ejecucion ordenada.

## Capas Del Sistema

### 1. Captura

Entrada rapida de ideas, conversaciones, documentos, clientes, oportunidades, problemas y decisiones.

Fuentes posibles:

- ChatGPT;
- Codex;
- Gmail;
- Drive;
- WhatsApp;
- reuniones;
- notas personales;
- Airtable;
- Excel;
- OneDrive.

### 2. Clasificacion

Cada entrada debe clasificarse para evitar acumulacion desordenada.

Categorias iniciales:

- idea;
- proyecto;
- cliente;
- proveedor;
- decision;
- tarea;
- alerta;
- documento;
- oportunidad;
- problema.

### 3. Enriquecimiento

La IA agrega contexto, resume, conecta con proyectos existentes, identifica riesgos y propone proximas acciones.

### 4. Decision

Cuando una accion requiere criterio ejecutivo, G-OS prepara una recomendacion y pide decision a Guillermo.

### 5. Ejecucion

Las tareas aprobadas se transforman en acciones, documentos, automatizaciones, comunicaciones o seguimiento.

### 6. Seguimiento

Cada elemento relevante debe tener estado, responsable, fecha, proxima accion y fuente de verdad.

### 7. Aprendizaje

Las decisiones, resultados y aprendizajes vuelven al sistema para mejorar futuras recomendaciones.

## Estados Base

- capturado;
- en analisis;
- pendiente de decision;
- aprobado;
- en ejecucion;
- bloqueado;
- completado;
- archivado.

## Reglas Operativas

- Separar datos maestros de datos transaccionales.
- Evitar duplicar informacion entre modulos.
- Registrar fechas, responsables y proximas acciones.
- Mantener documentos legibles por personas y agentes IA.
- Priorizar claridad antes que volumen.
- Escalar de documento a base de datos solo cuando el proceso lo justifique.
